# Refactoring Patterns

- Extract Function: split long or duplicated logic.
- Inline Function: remove tiny wrappers that hide obvious behavior.
- Rename: improve clarity of names.
- Move: relocate logic to the correct module.
- Extract Variable: name a complex expression.
- Inline Variable: remove unnecessary aliases.
- Split Phase: separate parse/validate/transform steps.
- Remove Dead Code: delete unused code after verification.
