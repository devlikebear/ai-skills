# Work Order Template

Use this format before direct implementation.

```markdown
## Goal
<one short paragraph>

## Non-goals
- <do not change>

## Touch points (<=5)
- <file1>
- <file2>

## Steps
- [ ] Step 1
- [ ] Step 2

## Acceptance criteria
- [ ] AC1
- [ ] AC2

## Verification commands
- <test/build/lint command>
```
